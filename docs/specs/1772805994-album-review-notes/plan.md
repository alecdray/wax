# Album Review Notes — Implementation Plan

## Approach

Add a `review` text column to the existing `album_ratings` table and expose it through a dedicated notes modal. The notes modal is opened via a separate icon button in the album row (next to the rating badge), keeping the rating flow unchanged. Notes are independent of ratings — a user can write notes without a numeric rating. Text is capped at 2000 characters, enforced at both the handler and DB level. This is v1: one editable text field, no append-only history.

Alternatives considered:
- Adding notes as a tab inside the existing rating modal: rejected because it couples two distinct interactions and makes the rating modal heavier.
- Reusing the existing `UpsertAlbumRating` query for notes too: accepted — the same COALESCE pattern handles partial updates cleanly.

## Implementation Steps

1. **Migration** — Create and apply the migration that adds `review text` to `album_ratings`.

2. **SQL query** — Update `UpsertAlbumRating` in `db/queries/album_ratings.sql` to include `review` in the INSERT column list and `COALESCE(EXCLUDED.review, review)` in the ON CONFLICT clause. Add a dedicated `UpsertAlbumReview` query that inserts with a null rating and sets review, using the same COALESCE pattern.

3. **`sqlc generate`** — Run to regenerate `album_ratings.sql.go` and `models.go`.

4. **DTO + service** — Add `Review *string` to `AlbumRatingDTO` and populate it from `model.Review` in `NewAlbumRatingDTOFromModel`. Add `UpdateReview(ctx, userId, albumId, review string) (*AlbumRatingDTO, error)` to `review.Service` using the new `UpsertAlbumReview` query.

5. **Templ components** — Create `src/internal/review/adapters/review_notes.templ` with:
   - `ReviewNotesModal(album library.AlbumDTO)` — modal with a textarea (maxlength 2000), pre-filled with existing notes if present, `hx-post` on submit.
   - `CloseReviewNotesModal()` — mirrors `CloseRatingModal`.
   - `AlbumNotesIcon(album library.AlbumDTO, isOobSwap bool)` — small icon button in the album row; filled icon if notes exist, outline if not. Triggers `hx-get` to open the modal.

6. **HTTP handlers** — Add to `review/adapters/http.go`:
   - `GetReviewNotes(w, r)` — GET, renders `ReviewNotesModal`.
   - `SubmitReviewNotes(w, r)` — POST, validates review length (<= 2000 chars), calls `reviewService.UpdateReview()`, renders `CloseReviewNotesModal()` + OOB `AlbumNotesIcon` swap.

7. **`templ generate`** — Run after editing `.templ` files.

8. **Dashboard template** — In `albumRow`, add `@AlbumNotesIcon(album, false)` next to `@AlbumRating`. The notes icon sits in the same `<td>` as the rating badge or in its own column (keep in rating cell to avoid adding a column).

9. **Route registration** — Register in `server.go`:
   ```
   GET  /app/review/notes   → reviewHandler.GetReviewNotes
   POST /app/review/notes   → reviewHandler.SubmitReviewNotes
   ```

## Database Changes

**New migration** (run `goose -dir db/migrations create album_ratings_review sql` then `task db/up`):

```sql
-- +goose Up
ALTER TABLE album_ratings ADD COLUMN review text;

-- +goose Down
-- SQLite does not support DROP COLUMN in older versions; migration is a no-op on rollback
-- or recreate table without the column if needed.
```

**Updated upsert query** — the existing `UpsertAlbumRating` gains `review` in the INSERT and `COALESCE(EXCLUDED.review, review)` in the ON CONFLICT SET clause. A new `UpsertAlbumReview` query handles the notes-only path.

## Testing

Manual verification:
1. Open the dashboard. An album without notes shows an outline notes icon; an album with notes shows a filled icon.
2. Click the notes icon. The modal opens pre-filled with existing notes (or empty).
3. Type notes and submit. The modal closes; the notes icon in the row updates via OOB swap.
4. Open the rating modal and submit a rating. Confirm the review text is preserved (COALESCE is working).
5. Open the notes modal and submit a review. Confirm the numeric rating is preserved.
6. Submit notes exceeding 2000 characters. Confirm the handler returns a 400 with an error message displayed inline.
7. Submit notes for an album with no existing `album_ratings` row. Confirm a new row is created with null rating and the review populated.

## Risks & Mitigations

- **COALESCE on both queries** — Easy to accidentally drop the review value when updating a rating. Both `UpsertAlbumRating` and `UpsertAlbumReview` must use `COALESCE` for their respective nullable columns. Review the generated SQL after `sqlc generate`.
- **SQLite ALTER TABLE** — SQLite supports `ADD COLUMN` since 3.1.3 so this is safe. However, rollback requires recreating the table. Document this in the migration comment.
- **2000 char limit** — Enforced in the HTTP handler (return 400 with inline error component). The DB CHECK constraint is optional given templ's auto-escaping and handler-level enforcement; skip it to keep the migration simple.
- **OOB swap target ID** — `AlbumNotesIcon` must render with a stable `id` matching what the dashboard initially renders. Follow the same pattern as `GetAlbumRatingID` — e.g., `album-notes-<albumId>`.
